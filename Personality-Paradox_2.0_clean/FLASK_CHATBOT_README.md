# Flask 版本 ChatBot 功能說明

## 功能概述

Flask 版本已成功整合 **人格分析諮商助手 (ChatBot)** 功能,提供與 Streamlit 版本相同的 AI 對話體驗。

## 新增功能

### 1. ChatBot 頁面 (`/chatbot`)

- **即時串流回應**：使用 Server-Sent Events (SSE) 技術實現打字機效果
- **個性化對話**：基於測驗結果 (MBTI、Big Five、星座、黑暗三角) 提供客製化建議
- **馬卡龍色系介面**：溫馨舒適的紫色漸層設計

### 2. User Persona 側邊欄

左側顯示使用者的完整人格檔案：
- MBTI 類型與暱稱（帶 emoji）
- Big Five 特質進度條
- 星座資訊
- 黑暗三角特質（如果有）

### 3. 對話管理功能

- **自動保存**：每次對話自動儲存到 `data/memory_MBTIResult(type='XXX').json`
- **歷史載入**：重新進入頁面時自動載入過往對話
- **手動操作**：
  - 🗑️ 清除對話：刪除所有對話記錄
  - 💾 保存：手動觸發保存功能

### 4. 建議問題

提供 4 個常見問題快速按鈕：
- 為什麼我的測驗結果會是這樣？
- 如何運用我的人格優勢？
- 這些特質在工作上會如何表現？
- 如何改善我的盲點？

## 技術實現

### 路由結構

```python
/chatbot                 # ChatBot 主頁面
/chatbot/stream         # SSE 串流端點 (POST)
/chatbot/clear          # 清除對話 (POST)
/chatbot/save           # 手動保存 (POST)
/chatbot/history        # 獲取歷史 (GET)
```

### SSE 串流流程

1. 前端發送用戶訊息到 `/chatbot/stream`
2. 後端調用 `ollama_client.generate_stream()` 生成回應
3. 使用 `stream_with_context()` 包裝生成器
4. 每個 chunk 以 `data: {json}\n\n` 格式發送
5. 完成後發送 `{done: true}` 信號
6. 自動保存到 session 和長期記憶

### 記憶架構

```json
{
  "mbti_type": "ENFJ",
  "last_updated": "2025-12-08T10:30:00",
  "chat_messages": [
    {
      "role": "user",
      "content": "我的優勢是什麼？",
      "timestamp": "2025-12-08T10:25:00"
    },
    {
      "role": "assistant",
      "content": "根據您的 ENFJ 類型...",
      "timestamp": "2025-12-08T10:25:05"
    }
  ]
}
```

## 使用流程

1. **完成測驗**：MBTI → Big Five → 星座 → 黑暗三角（可選）
2. **查看結果**：在結果頁面點擊「💬 與 AI 諮商助手對話」
3. **開始對話**：
   - 點擊建議問題快速開始
   - 或在輸入框輸入自訂問題
   - 按 Enter 或點擊發送按鈕
4. **查看回應**：AI 助手會以打字動畫效果即時回應
5. **管理對話**：
   - 對話自動保存，下次進入會載入
   - 可隨時清除或手動保存

## 疑難排解

### 問題：SSE 串流無法連接

**解決方案**：
- 確認 Ollama 服務正在運行：`ollama serve`
- 檢查 gemma3:4b 模型已下載：`ollama pull gemma3:4b`

### 問題：對話歷史無法載入

**解決方案**：
- 檢查 `data/` 目錄是否存在
- 確認檔案權限正確
- 查看 console 日誌錯誤訊息

### 問題：打字動畫卡頓

**解決方案**：
- 檢查網路連線
- 降低 Ollama 模型參數（如有設定）
- 確認瀏覽器支援 EventSource
